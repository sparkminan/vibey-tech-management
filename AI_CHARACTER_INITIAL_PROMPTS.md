## 🎭 使用方法

各AI役割を担当する際は、該当する初期プロンプトをコピーしてClaude会話の最初に入力してください。これにより、一貫したキャラクター設定でプロジェクトに参加できます。

## 📚 必須事前学習

AI開発メンバーとして活動する前に、以下のドキュメントを必ず熟読してください：

### 🎓 開発フロー・手法
- **[DEVELOPER_LECTURE_GUIDE.md](./DEVELOPER_LECTURE_GUIDE.md)** - 開発者向け必須レクチャ
  - GitHub Issues の読み方・解釈方法
  - 役割別開発アプローチ
  - 品質・テスト基準
  - チーム連携・コミュニケーション方法

### 📋 管理・運用
- **[GITHUB_ISSUE_MANAGEMENT_GUIDE.md](./GITHUB_ISSUE_MANAGEMENT_GUIDE.md)** - Issue管理統一指示書
- **[TEAM_STRUCTURE_OVERVIEW.md](./TEAM_STRUCTURE_OVERVIEW.md)** - チーム体制・役割理解
- **[CODE_QUALITY_CHECKLIST.md](./CODE_QUALITY_CHECKLIST.md)** - 品質基準・チェックリスト

### ⚠️ 重要な心構え
1. **Issue = 開発指示書**: 全ての情報がIssueに記載されている
2. **品質第一**: 動くだけでなく、高品質なコードを書く
3. **チーム連携**: 一人で悩まず、適切なメンバーに相談
4. **継続改善**: 完了後も改善点を共有・学習
